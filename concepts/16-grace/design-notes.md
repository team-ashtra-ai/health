Concept name:
16 — Grace

Assigned inspiration URL:
https://beauty365.com.br/

What was studied:
The concept studies the reference as a structural prompt for service-commerce shelves: navigation hierarchy, first-screen rhythm, service grouping, mobile menu posture, footer density and premium pacing. No protected text, brand assets, code or photography from the reference is used.

How the header differs from the other concepts:
This concept uses a shop-like service nav. The header is assigned the unique runtime marker `header-16-grace` and a concept-specific CSS composition.

How the hero differs from the other concepts:
The hero is shaped around service-commerce shelves, with concept-specific grid, image treatment, radius, motion timing and headline structure.

How the page layout differs from the other concepts:
Page sections use a unique section order marker, component grid rhythm and panel mix generated for Grace. The flat pages are not routed through shared root templates.

How the mobile menu differs from the other concepts:
The mobile menu uses a sliding product menu with local JavaScript in `js/main.js` and local markup in `partials/mobile-menu.html`.

How the footer differs from the other concepts:
The footer uses a membership footer, local contact hierarchy and concept-specific footer marker `footer-16-grace`.

How the motion differs from the other concepts:
Motion is based on shelf hover lift. The local `main.js` sets unique menu and reveal behaviour for this concept.

How Sofiati’s brand identity was applied:
Sage green, ivory, cream, bronze/champagne accents, the Sofiati logo system, FS monogram assets, botanical imagery, clinical calm and responsible advanced aesthetic biomedicine language are applied throughout.

Why this concept is not a clone of the others:
It has its own folder, flat HTML pages, `css/style.css`, `js/main.js`, partials, copied assets, design notes, header marker, footer marker, mobile menu marker, hero structure, section order and interaction mode. It does not depend on root `/css`, `/js`, `/partials` or `/assets` at runtime.
