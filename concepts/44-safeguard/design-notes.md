Concept name:
44 — Safeguard

Assigned inspiration URL:
https://www.lamer.com.br/

What was studied:
The concept studies the reference as a structural prompt for cream skincare commerce: navigation hierarchy, first-screen rhythm, service grouping, mobile menu posture, footer density and premium pacing. No protected text, brand assets, code or photography from the reference is used.

How the header differs from the other concepts:
This concept uses a cream commerce header. The header is assigned the unique runtime marker `header-44-safeguard` and a concept-specific CSS composition.

How the hero differs from the other concepts:
The hero is shaped around cream skincare commerce, with concept-specific grid, image treatment, radius, motion timing and headline structure.

How the page layout differs from the other concepts:
Page sections use a unique section order marker, homepage index rhythm and panel mix generated for Safeguard. The HTML pages are slim shells with comments and local partial mounts; they are not routed through shared root templates.

How the mobile menu differs from the other concepts:
The mobile menu uses a cream category menu with local JavaScript in `js/main.js`, local partial loading in `js/partials.js` and local markup in `partials/mobile-menu.html`.

How the footer differs from the other concepts:
The footer uses a commerce footer, local contact hierarchy and concept-specific footer marker `footer-44-safeguard`.

How the motion differs from the other concepts:
Motion is based on cream layer motion. The local `main.js` waits for `js/partials.js`, then sets unique menu, reveal, form, floating WhatsApp and back-to-top behaviour for this concept.

How Sofiati’s brand identity was applied:
Sage green, ivory, cream, bronze/champagne accents, the Sofiati logo system, FS monogram assets, botanical imagery, clinical calm and responsible advanced aesthetic biomedicine language are applied throughout. English remains the source copy in the files, while the local concept JavaScript loads the public pages in Portuguese by default.

Why this concept is not a clone of the others:
It has its own folder, flat HTML page shells, `css/style.css`, `js/main.js`, `js/partials.js`, head/schema/header/navigation/mobile-menu/footer/form/widget partials, copied assets, design notes, unique markers, expanded homepage sections, language switcher and interaction mode. It does not depend on root `/css`, `/js`, `/partials` or `/assets` at runtime.
