# Partials Refactor Plan

Date: 2026-06-26

## Refactor Principle

Standardize technical skeletons, not visual personality. Partial names, mount points, ARIA roles, button states and data hooks should be predictable. Header shape, footer composition, menu reveal, CTA styling and decorative rhythm should remain concept-specific.

## Safe Implementation Rules

- Edit one concept first.
- After the POC, decide whether a small helper script is warranted.
- Do not normalize visual class names into one shared theme.
- Keep all old partial names until all pages are verified.
- If a partial is unused, mark it in cleanup docs first; do not delete it.
- After partial edits, run at least static link/a11y checks plus screenshots for edited concepts.
