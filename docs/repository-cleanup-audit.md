# Repository cleanup audit

Audit date: 2026-07-18. Baseline branch: `main` at `3c59bb057` (`PERFECT BEFORE CLEANUP`). Cleanup work is isolated on `chore/repository-cleanup`.

## Scope and deployment

This is a framework-free static bilingual site. Source pages are served locally with `python3 -m http.server 4173`; public shared chrome is loaded at runtime from `partials/`. `scripts/build/build-production.mjs` creates the deployment artifact in `dist/` and deliberately excludes authoring, QA and historical directories. No Cloudflare-specific configuration file or GitHub Actions workflow is tracked; deployment integration is therefore external to this checkout and must not be inferred or replaced.

At the time of this baseline, English pages were also represented by a stale generated-content pipeline. That duplicate pipeline has since been retired: current root English HTML is now the source of public page content. Journal pages are generated by `scripts/build-journal.py`; `pt/` and `partials/pt-BR/` are generated/localized by `scripts/generate-portuguese-site.py`. Portuguese output is not cleanup-only duplicate content.

## Deletion evidence method

Every deletion in this cleanup must satisfy all of: no public HTML reference; no CSS `url()` reference; no JavaScript, Python, Node, JSON, manifest, sitemap, structured-data or partial reference; no build/deployment reference; and no observed runtime request. Filename search is not the sole test. Asset byte equality alone is never deletion evidence for production assets.

## Open risks before structural cleanup

1. The committed generated CSS and Journal output are stale against their sources; regenerating them would be a content/visual change and needs its own approved visual-diff phase.
2. The interaction QA script cannot run with the declared dependencies (`playwright` vs `playwright-core`).
3. Existing CSS architecture warnings are real maintenance debt, but broad selector or `!important` removal cannot be proven visually neutral until the baseline tooling is fully operational.
