# Style Guide

Keep the visual identity calm, editorial, botanical, and consultation-first.

- Preserve the existing Sofiati palette, typography rhythm, spacing, and fine-line visual language.
- Use clear, expectation-led language. Avoid unsupported certainty, cures, invented testimonials, and before/after claims.
- Keep shared layout changes in modular CSS files when possible. Preserve production page styling in `css/pages.css`.
- Keep root pages flat and easy to edit in VS Code.
- Place reusable interface behavior in the matching `js/` module.
